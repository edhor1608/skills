# Obsidian Project Sync

Convert scattered project context into structured Obsidian docs.

## Target Structure

- One folder per project
- Core docs per project:
  - README.md
  - roadmap.md
  - backlog.md
  - decisions.md
  - notes.md

## Workflow

1. Discover candidate projects and source material.
2. Merge overlapping project names and aliases.
3. Create or update per-project folder structure.
4. Populate docs with high-signal content only.
5. Add bidirectional links across related projects.
6. Write a workspace index with project summaries.

## Guardrails

- Preserve existing user-authored notes.
- Mark inferred information clearly.
- Keep language consistent within each workspace.

## Output Format

- Updated projects
- New files created
- Files enriched
- Ambiguities requiring user decision
